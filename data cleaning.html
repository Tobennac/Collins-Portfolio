Project Overview

This project involved cleaning and preparing a housing dataset to ensure data integrity and reliability. The main task included date format conversion, handling empty fields by setting them to 'NULL', populating missing porperty addresses, and removing duplicate records. check out the script on my Github repository.

GOAL

The objective of this project was to clean and prepare the Nashville housing dataset for analysis by handling date conversions, removing empty fields, populating missing addresses, and eliminating duplicate records

RESULT

The dataset was successfully cleaned by converting date fields, filling in missing property addresses, setting empty fields to `NULL`, and removing duplicates. This enhanced the data's integrity and reliability for subsequent analysis.

PROJECT DURATION

The project was completed over a span of 2 days.

Date Conversion And Handling Empty Fields

The ‘SaleDate” column was originally in a non-date format. To ensure consistency and facilitate data-related operations, I converted the field to a proper date format using the CONVERT function. Also, several fields contained empty string, which can cause issues during analysis. I used the NULLIF function to replace these empty strings with NULL, a more appropriate representation for missing data in SQL.

Populating Missing Values

For records where the PropertyAddress field was empty or null, I used a join operation to find matching ParcleID values and populate the PropertyAddress field with corresponding values from other records. The COALESCE and NULLIF functions were used to ensure that the most accurate addressed were filled in.
